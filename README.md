# AI Task Worker

这是一个给人类配合 AI 使用的自动化编程 Worker。

它做的事情很简单：

- 持续扫描外部需求来源
- 找出需要处理的事项
- 交给 AI 编程工具去修改目标工程
- 记录扫描与处理过程

它不是一个“让人手动研究内部字段和流程”的工具，而是一个“让人通过 Codex 或 Claude Code 去驱动自动化处理”的工具。

## 它能做什么

这个 Worker 的核心价值是把“需求进来”到“AI 去改代码”之间的流程串起来。

你可以把它理解为一个中间层：

- 前面接需求来源
- 后面接 AI 编程工具
- 中间负责调度、执行、验证、记录

当前已经支持：

- 需求来源：Feishu 多维表格
- AI 执行器：Codex、Claude Code
- 运行记录：按来源分类记录扫描日志和处理日志

这个结构也为以后扩展留好了口子，比如：

- 新的来源可以是 nginx 日志、工单系统、Sentry、数据库、消息队列
- 新的执行器可以是更多 AI 编程工具

## 怎么用

你不需要先读懂全部代码，再去手动操作内部模块。

更自然的使用方式是：

- 在这个目录里直接运行 `codex` 或 `claude`
- 直接告诉 AI 你想监听什么、想让它处理哪个工程
- 让 AI 帮你追问缺失信息、生成来源配置、排查扫描、查看日志、扩展能力

现在配置也在往这个方向演进：

- 人类优先写“意图”
- 程序运行时使用结构化配置

也就是说，人更适合写接近说明书一样的来源意图，程序会直接读取它；如果你需要导出结构化配置，再额外编译。

换句话说，这个项目不是希望人类自己去记住所有内部结构，而是希望人类通过 AI 来操作它。

如果你不知道该怎么开始，可以直接这样说：

- `我想让这个项目监听飞书多维表格里的 bug 或需求，并自动去修改 /root/work/demo 的代码。你先告诉我需要准备哪些信息，如果缺什么再继续问我。`
- `我想给 /root/work/demo 增加一个自动处理需求的来源，你帮我设计并接入。`
- `帮我看看这个项目现在接了哪些来源，今天处理了什么。`

日常使用时，更推荐直接在这个目录里启动 `codex` 或 `claude`。  
具体命令和脚本不用在 README 里展开，AI 或人都可以直接查看 [package.json](/root/work/ai-task-worker/package.json)。

## 日志

运行日志会按来源分类落盘，便于你每天查看系统到底扫了什么、处理了什么。

日志目录大致是：

```text
logs/
  <source-name>/
    scan-YYYY-MM-DD.jsonl
    issues-YYYY-MM-DD.jsonl
  system/
    worker-YYYY-MM-DD.jsonl
```

含义如下：

- `scan-*.jsonl`
  记录每轮扫描摘要，适合看“今天扫了多少轮、每轮有没有命中”
- `issues-*.jsonl`
  记录真正发生动作的需求事件，适合看“处理了哪些事项、结果怎样”
- `system/worker-*.jsonl`
  记录 Worker 启动、轮次开始/结束、全局错误

设计上刻意避免“空扫描刷满日志”：

- 每轮扫描会有摘要
- 只有真正发生处理动作时，才会写 issue 事件明细

## 人类为什么要配合 AI 使用它

这个项目的目标，不是让人类手工维护一堆来源字段、执行出口和内部流程。

真正想解决的问题是：

- 人不想反复熟悉代码结构
- 人不想每次扩展来源都从头摸索
- 人希望直接对 AI 说“帮我接一个来源”“帮我切到 Codex”“帮我排查为什么没扫到”

所以这个仓库除了代码，也开始包含面向 AI 的规则文件，让 Codex / Claude Code 更容易理解：

- 这个项目的目的是什么
- 什么是来源，什么是执行器
- 哪些地方是扩展点
- 出问题先看哪里

## README 的定位

这个 README 面向人类使用者。

它重点说明：

- 这个程序能做什么
- 人类怎么借助 Codex 或 Claude Code 去使用它
- 运行后去哪里看结果

它不刻意展开内部字段、来源细节和底层实现，因为那些更适合交给 AI 代理在仓库内规则中理解和处理。

当前更推荐的流程是：

1. 在仓库目录运行 `codex` 或 `claude`
2. 直接描述你想监听什么、处理什么
3. 让 AI 继续追问缺失信息
4. 由 AI 生成或修改来源 intent
5. 再运行 Worker 或调试命令

如果你想把意图导出成结构化 JSON 留档或排查，再执行：

6. `npm run compile:intents`

## 给 AI 的规则

如果你希望 Codex 或 Claude Code 更稳定地操作这个项目，可以让它们先看：

- [AGENTS.md](AGENTS.md)
- [CLAUDE.md](CLAUDE.md)

这些文件不是给人类背内部实现的，而是给 AI 理解这个项目的边界、结构和推荐操作方式的。
